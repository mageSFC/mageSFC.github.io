Docker源码学习之docker-names-generator

发表于2023-09-27|更新于2023-09-27|docker
|阅读量:

docker源码学习之容器的随机名称的生成

首先是 发送 的 post 请求，被这个函数先处理

postContainersCreate() 函数

在这个函数里面 会从请求参数中获取到 name 的值，这里是我们传入 的 my， 也对应上 命令行参数 –name my

s.backend.ContainerCreate() 函数

调用这个函数的时候，第二个参数，里面有个属性 Name 就是 我们传入的容器名称。

daemon.containerCreate() 函数

daemon.create(ctx, &daemonCfg.Config, opts) 函数

daemon.newContainer(opts.params.Name,。。。。) 函数

到这里 真正 发现 使用 .Name 这个属性了。

daemon.generateIDAndName(name)函数

这里真正的 把 name 又传入了另外一个函数，从函数名称上就能看出来了。

这个函数里面 又3个小的函数

id := stringid.GenerateRandomID()

这里生成随机的字符串，作为 id。

1️⃣调用 rand.Read(b); 生成 32 自己的数据

2️⃣hex.EncodeToString(b) 字节转成 字符串

最后就是会生成 这样的 值： be94f7e505ecc06cd0d16dc0e0358ba5d33d04f469ebb56bbda60d3457458f60

daemon.generateAndReserveName(id)

来自 pkg/namesgenerator/names-generator.go 文件

如果传入的 name 是空字符串，这里就会 随机的 取个名称的

1️⃣ 调用 namesgenerator.GetRandomName(i)获取个名称。

GetRandomName()里面也没什么神秘的， 就是定义了 2个 切片，

一个切片保存了很多形容词，另一个切片保存了很多人名。一些科学家，和 黑客的 名字。

上面 那么多行 就是 预定义的 2个切片，

daemon.reserveName(id, name)

注册 name 和 id， 如果名称存在了，还会继续尝试 6 次 。

容器名称应该是 全局唯一的。不能重复的。容器ID 更不用说了，必须唯一。

return id, name, nil

返回容器ID 和 容器name。
