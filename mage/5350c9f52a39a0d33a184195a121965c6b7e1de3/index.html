Jenkins插件学习之extended-choice-parameter-plugin

发表于2023-08-06|更新于2023-08-06|jenkins
|阅读量:

今天介绍一个插件 extended-choice-parameter-plugin

另外一个 扩展的选项 参数

  1. JSON Parameter Type 举例

参考 这里：
https://wiki.jenkins-ci.org/JENKINS/Extended-Choice-Parameter-plugin.html

我自己魔改的

  1. 仿照extensible-choice-parameter-plugin插件修改的。提供了几个ChoiceListProvider类型的class供选择。

    • BasicChoiceListProvider ，

    • MultilevelChoiceListProvider ， 

    • JSONChoiceListProvider

  2. 其中第一个 BasicChoiceListProvider 会用的比较多。可以有 单选，多选，checkbox，radiobox，hidden 这些样式。

    • Single Select: user chooses a single selection from a drop down menu, populated by either explicit values (see Value field below) or a property file (see Property File and Property Key fields below)单选：用户从下拉菜单中选择一个选项，由显式值（参见下面的值字段）或属性文件（参见下面的属性文件和属性键字段）填充

    • Multi Select: a user can choose multiple selections from a multi-line box, populated by either explicit values (see Value field below) or a property file (see Property File and Property Key fields below)

    • Check Boxes: user can check off zero or more check boxes, labeled by either explicit values (see Value field below) or a property file (see Property File and Property Key fields below)

    • Radio Buttons:user can check off zero or only one.

    • Text Box: like a input label. user cank type or modify the value. 文本框，这个就和 普通的 string 类型的 参数一样，可以手动输入文本字符串。唯一多的一点地方是配置默认值，
      默认值可以来自 “Default Value”， “Default Property File”， “Default Groovy Script”，“Default Groovy Script File”

    • Hidden: like a input label with hidden, input type=”hidden” 这个类型和 这个 插件 hidden-parameter-plugin 实现的功能类似，不同的地方也是在配置默认值。

https://github.com/mamh-java/extended-choice-parameter-plugin/tree/extended-choice-parameter-0.83-for-jenkins-2.249.3
